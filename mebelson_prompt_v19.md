# MEBELSON® — МП Аналитика по конкурентам (v19)

## Контекст
Система автоматического конкурентного анализа маркетплейса Ozon для MEBELSON® (мебельная фабрика, Ижевск). Цель — находить товарные возможности: что запустить, что усилить, где белые пятна.

## Методология — три группы
- **Группа 1 — Белые пятна:** продавец ≠ MEBELSON + оборот ≥ порога категории за 28 дней
- **Группа 2 — Новинки:** карточка ≤ 90 дней + оборот ≥ порога категории
- **Группа 3 — Дойные коровы:** карточка ≥ 365 дней + дней без остатка ≤ 3 + оборот ≥ порога категории
- Исключаем: металл, массив, шпон — не производим

## Фильтр маржинальности (актуальная версия, 30.06.2026)
₽/кг = Цена ÷ (Объём, л × K), порог ≥ 300 ₽/кг

K — коэффициент объёмного веса, свой для каждой категории. Выведен из фактического веса с упаковкой по 20–30 карточкам конкурентов на категорию (медиана выборки). Старая универсальная формула объём × 0.75 признана неверной (ошибка ~155%) и больше не используется.

**Проверенные K-коэффициенты:**
| Категория | K | % проходят порог 300 ₽/кг |
|---|---|---|
| Туалетный столик | 0.358 | 42% — здоровая ниша (свежий Скрипт 1: 29 из 69) |
| Комплект мебели для прихожей | 0.255 | 0% — средний сегмент не сформирован на Ozon |
| Шкаф распашной | 0.450 | 4% — средний сегмент не сформирован |
| Компьютерный стол | 0.264 | 55% — здоровая ниша |
| Письменный стол | 0.376 | 10% |
| Тумба | 0.378 | задан в скрипте (CATEGORY_K), валидация в очереди |
| Комод | 0.370 | задан в скрипте (CATEGORY_K), валидация в очереди · медиана 242, проходят 31% |
| Все прочие категории | 0.294 (default, не валидирован) | — |

**Правило валидации K:** валидируем по мере приоритета (20–30 карточек на категорию). На неприоритетных категориях осознанно берём дефолт 0.294 — ради скорости, с пометкой «не валидирован» на дашборде.
**В очереди на валидацию (🔴 Топ, K не проверен):** Тумба, Комод, Обувница, Двуспальная кровать, Шкаф навесной.

Высокий % выбросов по ₽/кг внутри категории не считается ошибкой данных — может означать готовность рынка платить за более полную комплектацию. Выбросы считаются по IQR×1.5 от медианы внутри своей категории и подсвечиваются в Excel.

## Источники данных
- **Ozon ЛК (основной):** Аналитика → Товары на Ozon → XLSX, период 28 дней. Все аналитические данные только отсюда
- **mpstats:** фото (колонка Thumb, джойн по SKU) и **CSV отзывов** по товарам. Оборот из mpstats НЕ используем — ненадёжен с апреля 2026

**Как тянуть рыночные данные из Ozon:**
- Произвольные интервалы недоступны — только пресеты: 7 д / 28 д / Квартал / Год. Панели (2-й слой) = **28 д**; приоритизация = **Квартал**.
- Колонка «Всего» = весь рынок категории. «Мой магазин» (если включена настройка) = ОДИН аккаунт (из которого выгрузка), НЕ 4 — для нашего оборота напрямую не использовать, только для перекрёстной сверки.
- Аналитика брендов: строка бренда «Mebelson» — фильтр по бренду. Число продавцов под брендом показывает, есть ли сторонние.

## Инсайт: бренд «Mebelson» на Ozon ≠ наши продажи
Строка бренда «Mebelson» в аналитике брендов Ozon показывает БОЛЬШЕ, чем наши реальные продажи по 4 аккаунтам:
- Тумба: бренд 11.5 млн (28 дн) · наши 4 акк. 8.72 млн (июнь)
- Компьютерный стол: бренд 7.75 млн · наши 5.51 млн · **5 продавцов** под брендом (у нас 4 → есть сторонние)
- Двуспальная кровать: бренд 3.76 млн · наши 2.18 млн · наш основной аккаунт = 56% бренда, остальное — сторонние
Под именем «Mebelson» на Ozon торгует кто-то ещё (перекупы/реселлеры), либо часть наших продаж идёт под «Бренд отсутствует».

Следствия:
- «Наш оборот / доля / место» НЕ берём из брендовой строки Ozon — только сумма 4 аккаунтов.
- «Место среди брендов» в панелях не считаем: наши 4 аккаунта не сводятся к одной строке бренда.
- **Блок расхождения — стандартный элемент панели** (где есть выгрузка брендов): серый, отделённый от KPI. Формат: «бренд Ozon (28 дн) = X · наши 4 акк. (месяц) = Y · мимо нас ≈ X−Y». Число продавцов подписывать.
- Отдельная задача: кто ещё продаёт под брендом «Mebelson» и в каком объёме.

## Правила работы в проекте
- Один чат = одна категория или одна задача
- Данные Ozon и WB никогда не смешиваем
- Оборот всегда из Ozon ЛК — никогда из mpstats
- Свой оборот MEBELSON — всегда сумма 4 магазинов (Mebelson + Адванта + Маркетсон + Щербаков), никогда один магазин и никогда брендовая строка Ozon
- **Перекрёстная сверка оборота:** сверять между отчётами — месячный файл 4 акк. vs Ozon «Мой магазин» vs строка бренда. При расхождении — проверять маппинг L3 (месячный файл местами недосчитывает; напр. кровати за 3 мес: 7.38 млн vs Ozon 8.75 млн)
- Периоды не смешивать: свой оборот — последний полный месяц; рынок Ozon — 28 дней; доля — к одному периоду, подписывать
- Панель собирать строго по КАНОНУ (9 блоков) ПОВЕРХ эталона — брать его блок `<style>` дословно (hero-градиент, .hkpi 6 KPI, .mb-card, .rkg-summary, .filter-btn чипы, .gallery/.card, .boli-card, .gip-card, .cj-flow, .vyvod-card, .dtbl), переносить данные категории в ту же структуру. Эталон = prihozhaya_panel.html
- Файлы данных (XLSX, CSV) загружаем вручную в каждый чат — не храним в проекте
- Никогда не писать про ассортимент MEBELSON без файла Товары — читать факты, не придумывать

## Второй скин — «Вид решений» (арт-директору) + генератор
Параллельно коммерческому порталу (`index.html`, скин 1) живёт **вид решений** (`index2.html`, скин 2) — тот же датасет, но verdict-first, чистый визуальный язык для арт-директора. Боевой адрес обоих скинов — **VPS** (`api.mebelson.ru/portal/`, логин `mebelson`, защита настроена); **GitHub** (`alisaalena/mebelson-portal`, приватный) — резервное хранилище/история версий (см. раздел «Портал»).

**Каждая категория во втором скине = ПАРА файлов (обе в одной папке с `index2.html`, самодостаточно, без зависимости от `galleries/`):**
- `<кат>_reshenie.html` — экран решения: вердикт (🟢/🟠 + слово) → герой-число (рынок) → метрики → уровень доверия (A/B, раскрытие «почему») → «Что делать» со сроками → Боли → CJ → Гипотезы → Доказательная база (журнал + ссылка на галерею). **Эталон = `reference-screen.html`** (письменный стол).
- `<кат>_galereya.html` — галерея ТОП-10 с якорями `id="lead1…lead10"`, фильтр-чипы по цене (+ «маржа ≥ 300» где есть ₽/кг), легенда цветов ₽/кг, крошки назад к решению и к списку. **Эталон = `pismennyy_stol_galereya.html`.**
- В блоке «Что делать» экрана решения подписи-ссылки ведут на карточку лидера: `<кат>_galereya.html#leadN` (привязка «действие → лид» по бренду, названному в действии).

**Генератор `gen_reshenie.py`** (ядро задачи 4): на вход блок данных категории — `DATA` (вердикт, метрики, доверие, действия, боли, CJ, гипотезы, журнал) + `LEADREFS` (действие → номер лида) + `GALMETA` (заголовок, медиана ₽/кг, K). Отдаёт ОБА файла. `<style>` берётся дословно из эталонов (нулевой дрейф). Картинки ТОП-10 берутся из `<кат>_panel.html` (парсер трёх разметок), пережимаются Pillow → JPEG q72 ≤920px → base64 на диске (не в переписку). ₽/кг на карточке — если он есть в панели категории (у старых панелей его может не быть → карточка без точки маржи).

**Активация во втором скине:** в `index2.html` строку категории перевести из «разбор готовится» (`.row soon`) в `.row live` со ссылкой на `<кат>_reshenie.html` + вердикт + цвет точки (🟢 sage / 🟠 amber). `--amber` в `:root` уже добавлен.
